## **WHAT CHANGES BASED ON CONTEXT**

### What Stays Flexible (Per Topic Need)
- **Which components** the topic uses
- **In what order** they appear
- **How many** of each type
- **Depth of each component**

### What Doesn't Change
- Every topic gets **all necessary components to teach that concept fully**
- No topic is "abbreviated" or "light" because it's later in the course
- No "review chapters" that skip important teaching blocks
- Quality of explanation is consistent throughout

## **THE RULE: "FULL COVERAGE PRINCIPLE"**

For any given topic, a teacher should ask:

**"What does a student need to FULLY understand this concept?"**

Then include those components. That's it.

## **FLEXIBLE COMPONENT PATTERNS (Not Hierarchy)**

### Pattern A: Conceptual Learning
*Used when introducing a concept from first principles*

```
1. Hook (optional - if there's a natural curiosity hook)
2. Explanation (core concept breakdown)
3. Definition (formal or simple, depending on rigor needed)
4. Real-World Example (bridge to practice)
5. Alert: Tip (helpful hint)
6. Checkpoint Quiz (quick validation)
```

### Pattern B: Problem-Solving Focus
*Used when topic is about solving problems or techniques*

```
1. Academic Definition (formal technique)
2. Explanation (how it works)
3. Guided Exercise (step-by-step walkthrough)
4. Real-World Example (when you'd use this)
5. Checkpoint Quiz (practice)
6. Alert: Common Mistake (watch out for this)
```

### Pattern C: Behavioral/Psychology Topic
*Used for organizational behavior, decision-making, etc.*

```
1. Hook (engaging human moment)
2. Explanation (theory)
3. Street Smart (casual, relatable explanation)
4. Real-World Example (story from business/life)
5. Alert: Tip (practical takeaway)
6. Checkpoint Quiz (did you get it?)
7. Deep Dive (research/theory rabbit hole)
```

### Pattern D: Complex Systems
*Used when topic has multiple interconnected parts*

```
1. Explanation (overview)
2. Academic Definition (define key terms)
3. Real-World Example (story showing how parts work)
4. Alert: Prerequisite (what you need to know first)
5. Guided Exercise (walk through the system)
6. Checkpoint Quiz (validate understanding)
7. Deep Dive (advanced implications)
8. Alert: Common Mistake (where students go wrong)
```

### Pattern E: Light Topic (Quick Concept)
*Used when topic is straightforward and doesn't need much depth*

```
1. Explanation (brief)
2. Definition (quick reference)
3. Real-World Example (one example)
4. Checkpoint Quiz (validate)
```

### Pattern F: Heavy Topic (Complex Deep Learning)
*Used when topic is dense and needs thorough treatment*

```
1. Hook (create context)
2. Academic Definition (formal)
3. Explanation (detailed breakdown)
4. Street Smart (accessible version)
5. Real-World Example (multiple examples)
6. Guided Exercise (walkthrough)
7. Checkpoint Quiz (mid-check)
8. Deep Dive (advanced analysis)
9. Alert: Tips (multiple tips)
10. Alert: Common Mistakes (multiple mistakes)
11. Guided Exercise (another approach)
12. Checkpoint Quiz (final check)
```

## **MEASURING TOPIC COMPLETENESS**

After building a topic, ask:

- [ ] Does a student understand the **core concept** fully?
- [ ] Does a student know when to **apply** it?
- [ ] Does a student know when **not** to apply it?
- [ ] Does a student understand **real-world context**?
- [ ] Does a student know **common mistakes**?
- [ ] Can a student **solve a problem** using this?
- [ ] Can a student **explain it casually** to a friend?
- [ ] Can a student **pass an exam** on this?

If you answer YES to all (or most) of these, your topic has Full Coverage. âœ…
